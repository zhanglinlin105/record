# MCP Host / Server / Client

## 三个角色

```
Host    用户直接使用的 AI 应用
Client  Host 内部负责和 Server 通信的组件
Server  对外暴露工具的服务
```

---

## 类比餐厅

```
Host   = 餐厅
Client = 服务员
Server = 厨房

用户坐在餐厅（Host）
服务员（Client）负责跑腿传递
厨房（Server）负责做菜

用户不直接进厨房
厨房不直接面对用户
服务员在中间传递
```

## 各自职责

### MCP Host

```
用户直接使用的应用

例子：
• Claude Code
• Cursor
• Claude 桌面版
• 你自己开发的 Agent

职责：
• 管理所有 MCP Client
• 和 AI 模型交互
• 把 MCP 工具转换成 FC 格式给模型
• 把模型的 FC 调用转换成 MCP 请求
• 展示结果给用户
```

---

### MCP Client

```
Host 内部的组件
每个 Server 对应一个 Client

职责：
• 和对应的 MCP Server 建立连接
• 获取 Server 暴露的工具列表
• 发送工具调用请求
• 接收工具执行结果
• 处理连接异常

用户看不见 Client
它在 Host 内部默默工作
```

---

### MCP Server

```
对外暴露工具能力的服务

例子：
• 文件读写 Server
• 搜索 Server
• 数据库 Server
• GitHub Server
• 天气 Server

职责：
• 暴露工具列表
• 接收工具调用请求
• 执行具体操作
• 返回结果

不需要知道是哪个 Host 在调用
只需要遵守 MCP 协议 ✅
```

## 具体例子：Cursor 查文件

```
用户在 Cursor 里说：
"帮我读取 config.json"

┌─────────────────────────────────────────┐
│           Cursor（Host）                │
│                                         │
│  1. 模型决定调用 read_file 工具         │
│  2. Host 找到对应的 Client              │
│                                         │
│   ┌──────────────────┐                  │
│   │   File Client    │                  │
│   └────────┬─────────┘                  │
└────────────┼────────────────────────────┘
             │ MCP 协议
             │ "调用 read_file"
             │ "参数: config.json"
┌────────────▼────────────────────────────┐
│         File MCP Server                 │
│                                         │
│  3. 接收请求                            │
│  4. 执行 fs.readFile("config.json")    │
│  5. 返回文件内容                        │
└─────────────────────────────────────────┘
             │ 返回结果
             ▼
        Cursor 拿到内容
        给模型继续处理
```
